# Schauwerk Visual System v2

Visual System v2 replaces the old count-heavy Miro quality interpretation with a declared, deterministic design contract.

## Core rule

The information function selects the object:

- frames carry narrative chapters;
- shapes carry entities, decisions, actions and risks;
- connectors carry typed relationships inside one frame;
- tables carry dense comparisons;
- documents carry explanations and sources;
- sticky notes carry only mutable, open input.

A finished fact rendered as a sticky note is a quality blocker. So are missing reading paths, missing title/thesis hierarchy, excessive visual coverage, connector clutter, semantic colour misuse and an unseparated evidence appendix.

## Board rhythm

A presentation-grade board starts with orientation, provides a system map, develops the content in bounded modules, produces a synthesis and ends with evidence. Each frame has one main claim and uses the same 1120 × 630 canvas, 20-pixel grid and 60-pixel safety margin.

The palette has five semantic roles: ink, structure, evidence, decision and risk. Colour is redundant with text and shape; it is never the only state cue.

## Quality model

The gate scores five equal dimensions:

1. architecture;
2. semantics;
3. hierarchy;
4. density;
5. consistency.

Release requires at least 90/100 and no blocker. The declared plan is the source for visual review because Miro readback may omit geometry. Remote readback proves creation and conformance only.

## Official Miro basis

The design is informed by Miro's official Canvas, template, presentation, diagram and Help Center material. In particular, it uses the distinction between diagrams, tables, timelines, documents and presentation frames, and follows Miro's own emphasis on focus, layers, full-screen presentation and modular handling of large boards.

## Human review binding

The automatic score is never an aesthetic verdict. `schauwerk visual review-v2` accepts a strict seven-axis human review input and binds it to the exact board digest, local quality receipt and remote item-type/connector counts. A PASS cannot contain a failed axis. Missing authenticated screenshot access must be stated explicitly; an unauthenticated private-access page cannot be used as visual evidence.

## Geometry contract

The board specification carries declared design boxes for overlap, whitespace and density review. Miro `TEXT`, `SHAPE` and `STICKY` primitives receive requested dimensions. Miro `TABLE` and `DOC` primitives are provider-auto-sized by the current layout DSL; their declared boxes are design intent, while live conformance verifies their exact type, count and anchor rather than claiming unavailable provider width or height. This limitation is explicit and cannot be converted into an automatic aesthetic claim.
